# 一、

1. **为什么要做？**（业务痛点、老板真实意图）
2. **为谁做？**（用户角色：产品运营 / 销售 / 客服 / 财务 等）
3. **解决什么问题？**（不能是 “提升效率”，要具体）
4. **边界在哪？**（一期做什么、不做什么）

输出物：
- 需求背景 & 目标
- 一期范围（MVP 边界）

---

# 二、业务梳理

顺序：

### 1. 业务流程

- 现在线下 / 旧系统是怎么走的？
- 谁发起、谁审批、谁操作、谁结束？
- 哪里最痛、最耗时、最容易错？

输出：**现状流程图**

### 2. 未来流程（理想态）

- 新系统要把流程改成什么样？
- 哪些步骤合并、自动化、去掉？

输出：**未来态流程图**

### 3. 角色与权限

- 有哪些角色？
- 每个角色能看什么、操作什么？

输出：**角色权限表**

### 4. 核心业务对象（数据模型）

B 端最关键：

- 订单 / 合同 / 客户 / 商品 / 库存 / 工单 / 报表…
- 它们之间是什么关系？

输出：**简易 ER 图 / 字段清单**

---

# 三、拆功能 → 排优先级

把流程变成功能点，例如：

- 新增 XX
- 编辑 XX
- 列表查询
- 导入导出
- 审批流
- 权限控制
- 报表统计

然后按：

**核心必须 → 重要 → 锦上添花**

划分 MVP 范围。

输出：

- **功能清单**
- **优先级矩阵**

---

# 四、确认方案，再进入原型

到这里，你才**具备画原型的资格**。

你需要先确认：

1. 业务逻辑没问题
2. 流程没问题
3. 范围没问题
4. 老板 / 业务方对齐没问题
